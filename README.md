# Open-source sea turtle population viability analysis and trend toolkit

This repository provides an open-access, stochastic simulation framework for sea turtle population assessments. The underlying statistical methods transition traditional deterministic evaluations into state-space modeling environments, directly supporting robust Endangered Species Act (ESA) Section 7 consultation frameworks.

The methodologies embedded here are based on the state-space architectures developed by **Martin et al. (2020)** and the Conway-Maxwell-Poisson (CMP) take distribution adaptations defined by **Siders et al. (2023)**.

---

## Prerequisites and installation

### 1. System dependency: JAGS
The trend estimation engine relies on **JAGS (Just Another Gibbs Sampler)** via the `jagsUI` package. You must have the core JAGS binary application installed on your operating system before running the R code:
* [Download and install JAGS](https://sourceforge.net/projects/mcmc-jags/)

### 2. R packages
Execute the following block in your R console to install the necessary processing, simulation, and visualization libraries:

```R
install.packages(c("dplyr", "tidyr", "ggplot2", "mvtnorm", "truncnorm", "jagsUI"))
```

## Repository directory

### Core JAGS models (/models)
* **`singleUQ.txt`**: A multivariate state-space trend model. It estimates a single, underlying, regionally shared trend ($U$) across multiple index nesting beaches while isolating process variance ($Q$) and site-specific observation errors ($R_j$).

### Execution scripts (/scripts)
* **`sea_turtle_pva.R`**: A single, monolithic execution script mapping fishery interactions to adult population dynamics. It contains an automated toggling system for user data integration alongside an internal mock data generator. This allows the trend engine, growth age-inversion tracker, and dual-scenario population viability loops to run completely out-of-the-box. All generated analysis figures and metric spreadsheets are routed into a local `/output` folder automatically generated by the script.

## How to adapt this toolkit for your own data

This framework can be applied to any marine megafauna population with nesting data and known interaction inputs. To transition this script to your target population, update the following structural components:

### 1. Data configuration toggle
Locate the user execution control block at the top of `sea_turtle_pva_engine.R`. Switch the parameter configuration to look like this:

```R
use_mock_data <- FALSE
```

Once disabled, uncomment the file path routing lines directly below it and update the string values to match your local CSV file directories:

```R
nest_file  <- "path/to/your/nest_counts_time_series.csv"
safe_file  <- "path/to/your/total_annual_expanded_take.csv"
obs_file   <- "path/to/your/individual_observer_measurements.csv"
```

### 2. Species-specific biological constants
Calibrate the life-history modifiers within the script's parameter block to align with your target species' published biology:

```R
sp_clutch_freq <- 3.0    # Average nests deposited per female per season
sp_remig_int   <- 2.5    # Average years between breeding seasons
sp_pf          <- 0.70   # Sex ratio modifier (proportion female)

# von Bertalanffy growth function parameters for size-to-age inversion
sp_linf        <- 95.0   # Asymptotic straight carapace length (cm)
sp_k           <- 0.12   # Growth coefficient
sp_tknot       <- -0.5   # Theoretical age at zero length
```

### 3. Fishery take distribution calibration
The impacts of incidental interactions are modeled via a Conway-Maxwell-Poisson (CMP) distribution to account for under- or over-dispersion caused by gear characteristics or regulatory interaction caps. Modify the baseline take metrics to match your specific fishery profile:

```R
cmp_mu <- 12.5           # Baseline anticipated interaction parameter
cmp_nu <- 0.15           # Dispersion parameter (nu < 1 = over-dispersed)
```
